# Audio Asset Operations contract

`/api/v1/audio-asset-operations` is a bounded, private Web-native utility for
an authenticated account's existing **Asset Vault** audio. It can inspect a
source, create a fixed MP3/M4A copy, or create one fixed loudness-normalized
M4A copy. It deliberately does not claim AI audio enhancement.

This is not a migration of the Bot's guarded `/audio_enhance` provider flow.
That Bot mapping remains guarded. This Web route owns a separate local,
deterministic, owner-scoped utility boundary.

## Scope and lifecycle

| Surface | Contract |
| --- | --- |
| `POST /api/v1/audio-asset-operations/inspect` | Probe one active private source. It completes without an output file. |
| `POST /api/v1/audio-asset-operations/convert` | Create a fixed MP3 or M4A copy of one active private source. |
| `POST /api/v1/audio-asset-operations/normalize` | Create a fixed M4A copy using the server-owned `speech_safe_v1` profile. |
| `GET /api/v1/audio-asset-operations` / `/{id}` | Owner-scoped metadata and lifecycle only. |
| `GET /api/v1/audio-asset-operations/{id}/download` | Owner-scoped output attachment only after a fresh verification. |

Lifecycle is `queued → processing → completed`. Input, source, runtime,
quota, storage or lifecycle errors become `failed`; a completed output that is
missing, corrupt or no longer matches its probe becomes `unavailable`.
`audio_inspect` is correctly `completed` with `output_available=false`: it
never pretends to create a file. Convert/normalize are accepted as completed
in the Portal only when their verified output is available.

The browser supplies only an Asset Vault UUID, a closed action/target choice
and an idempotency key. It cannot supply bytes, paths, URLs, hashes, storage
keys, file names, codec settings, gain, FFmpeg arguments or normalization
profile values.

## Supported media boundary

- Active owner-scoped source pairs are MP3/`audio/mpeg`, WAV/`audio/wav`,
  M4A/`audio/mp4`, and OGG/`audio/ogg`.
- Source is at most **25 MiB**, duration is **0.25–600 seconds**, sample rate
  is **8–48 kHz**, and it has one or two channels.
- A fixed transform target is only MP3 (`audio/mpeg`) or M4A (`audio/mp4`).
  A normalization always produces M4A using `speech_safe_v1`; it is not a
  claim of mastering, content safety, speech recognition or quality repair.
- Output is at most **12 MiB**. Server-side ffprobe checks duration, channels,
  sample rate, container and codec before an output is marked available.

## Private storage and delivery

1. The server opens only an active, owner-scoped Asset Vault source through a
   descriptor-pinned stream, then verifies size, digest, key and source
   lifecycle revision before completion.
2. FFmpeg/ffprobe run only with server-owned fixed arguments, bounded timeout
   and shared local media capacity. Browser code never processes audio.
3. Generated bytes use a server-owned filename in a dedicated private
   `outputs/` root. The server rehashes and re-probes before and after atomic
   publication; reconciliation fails interrupted work and removes orphaned
   staging/output safely.
4. Every list/detail rechecks a candidate output. Download re-pins and
   rehashes/re-probes it into an anonymous sealed attachment with `private,
   no-store`, `nosniff`, `no-referrer`, `sandbox` and
   `Cross-Origin-Resource-Policy: same-origin`.
5. The output root must be distinct from Asset Vault, Project Packages,
   Document/Image/Subtitle/Video operation roots and `/static`; production
   requires a persistent-volume child directory.

## Configuration and protection

The feature is disabled by default. Enable it only after installing and
pinning locally trusted FFmpeg/ffprobe binaries and attesting that this
SQLite executor has exactly one replica:

```text
WEBAPP_ASSET_VAULT_ENABLED=true
WEBAPP_ASSET_VAULT_ROOT=/data/toanaas_webapp_assets
WEBAPP_AUDIO_ASSET_OPERATIONS_ENABLED=true
WEBAPP_AUDIO_ASSET_OPERATIONS_ROOT=/data/toanaas_webapp_audio_asset_operations
WEBAPP_AUDIO_ASSET_OPERATIONS_TOPOLOGY=sqlite_single_replica
WEBAPP_REPLICA_COUNT=1
WEBAPP_AUDIO_ASSET_OPERATIONS_QUOTA_MB=64
WEBAPP_AUDIO_ASSET_OPERATIONS_FFMPEG_BIN=/usr/bin/ffmpeg
WEBAPP_AUDIO_ASSET_OPERATIONS_FFPROBE_BIN=/usr/bin/ffprobe
```

Signed session ownership, CSRF on writes, bounded raw bodies, independent
rate limits, immutable idempotency receipts, account output quota, audit
events and transition history apply to every operation. Generic Jobs/Assets
may project a transform only when its typed verification remains true; an
inspect record is outputless and does not become a fake downloadable asset.

## Portal route and browser contract

The dedicated customer route is `/audio/assets`, under **AI Labs & Media**.
It is separate from music, voice, audio upload and Bot audio menus.

- Reads require a signed account, Asset Vault and the explicit audio flag.
  The Portal fetches only a typed active Audio Asset Vault selector and the
  narrow operations history, both `no-store`; it never falls back to generic
  Jobs, generic Assets, Bot state or browser cache.
- Metadata hydration is a Portal read state, not a transform lifecycle. While
  it loads, the source selector, pager and write action remain locked. Failed
  reads clear old projections instead of showing stale data.
- Route/session/epoch checks fence in-flight responses. Browser state contains
  only bounded display metadata and UUIDs; no source bytes, output URL, path,
  key or digest is persisted.
- A private download is a same-origin attachment fetch. The Portal validates
  attachment, MIME, `Content-Length`, `no-store`, `nosniff`, `no-referrer` and
  same-origin CORP headers before creating a temporary object URL, then
  revokes it after handoff.

## Explicit non-goals

- No Bot/Core Bridge call, Telegram state/upload/download, provider, paid
  API, Key4U call, AI music, TTS, voice clone, ASR, translate, dubbing,
  waveform/player/preview, browser FFmpeg or URL fetch.
- No canonical Bot job, worker, webhook, wallet/Xu, PayOS, refund, payment,
  package, provider-cost or production deployment action.
- No public URL, static download, stream, PWA/private cache, raw upload from
  this route, user-controlled codec/FFmpeg parameters or fake completed output.
