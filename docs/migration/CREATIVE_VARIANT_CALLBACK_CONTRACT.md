# Creative variant callback contract

The frozen Bot owns the `creative|*` callback handler. It is a Telegram-admin creative-selection mutation, not a Web Content Studio action: the handler requires the configured Telegram admin, accepts only `creative|select|<variant-id>`, resolves an opaque Bot creative-variant identifier, selects it, clears sibling selections, updates the linked production job to script/working state, and renders a Telegram handoff instruction. A callback is not a Web creative record, admin permission, browser back/reset, workflow selection, job state, provider action, output authorization or delivery contract.

| Frozen Bot callback family | Web target/boundary | Audit resolution | Required boundary |
| --- | --- | --- | --- |
| all frozen creative\|select\|* literals and templates | CREATIVE_VARIANT_SOURCE_REVIEW_REQUIRED | creative_variant_callback_requires_web_native_owner_role_contract | Telegram admin plus opaque Bot creative-variant ID; selects one canonical variant, clears sibling selection and updates production job script/working state |
| case variants, missing tokens, suffixes or future creative\|* values | CREATIVE_VARIANT_SOURCE_REVIEW_REQUIRED | creative_variant_callback_requires_web_native_owner_role_contract | no Web Content Studio/Admin route, browser navigation/reset, Bot creative/job/handoff replay, provider/output/delivery/ledger action or runtime claim |

Every `creative|*` source remains `CREATIVE_VARIANT_SOURCE_REVIEW_REQUIRED` until a workflow-specific Web-native owner/role contract exists. It cannot open `/content-studio` or an Admin route; navigate/reset the browser; receive/replay a Bot creative variant ID, selected-state mutation, linked production-job update or handoff instruction; invoke a provider; expose an output; send a delivery; mutate a payment/refund/ledger record; or claim completion. A future Web Creative Studio must start from its own signed staff session, server-side role checks, CSRF, confirmation/idempotency/audit rules for writes, and independently owned Web creative/work records or a separately reviewed redacted bridge/read-model. It must never accept or replay a Bot callback or Telegram creative state.
