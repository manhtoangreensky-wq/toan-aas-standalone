# Account Activity boundary

`/account/activity` is an owner-scoped history of activity generated by the
standalone Web App itself. It is intentionally available to a signed account
before Telegram linking, because it does not need a Bot/Core Bridge read.

## Contract

- `GET /api/v1/account/activity` reads at most 50 rows where
  `web_audit_events.account_id` equals the signed session's account.
- The response exposes only a server-normalized `label`, `category`, public
  `status`, and `created_at`.
- The query never selects or returns raw `request_id`, `target`, `detail`,
  `canonical_user_id`, email, session ID, token, password, provider payload,
  payment information, wallet amount, job output, or ticket content.
- Unknown actions are collapsed to a generic Web activity label. Admin actions
  are also generic; this route is not an Admin audit export.

## Non-goals

This page is not a Bot history, Xu ledger, PayOS history, job/asset delivery
history, provider log, ticket thread, OAuth token history, or canonical
Telegram identity viewer. Those sources remain behind their own owner-scoped
canonical adapters and are never copied into this Web-owned audit projection.

The table remains append-only and the composite account/time index is
additive. Reading it does not create an audit event, call a provider, or call
the private Bot bridge.
